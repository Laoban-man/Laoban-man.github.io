Interact with API and dashboards

Datascience is only part of the data pipeline. Once data analysis is done and prediction models have been established, nothing quite works as well at communicating \ how a model works than visualizing it in action. A standard POC would enable fake users to interact with an API capable of accessing a proprietary model \ and present prediction results as well as explain how the model worked.

Beyond the machine learning algorithm selected, this project focuses on implementing such a POC by designing an API, hosting it, the trained model and training data on a cloud server \ accessible online and visualizing on a webapp hosted by Heroku. (While I am a fan of the ease of use of the Heroku webapp service, I rather prefer setting up servers from scratch.)

Hug API

There are quite a few alternatives to choose from, perhaps the most popular one being FastAPI actually, but the HUG API framework is interesting in its simplicity and argued speed versus other frameworks. Here, we use the HUG API to experiment and while the documentation may not be comprehensive, the framework is remarkably simple.

Dash Plotly Dashboard

Again, Streamlit may be a more popular Python dashboarding framework currently, but Dash Plotly is more developed and better yet, it available in Python and R (and potentially Julia, and up and coming datascience languages.)

To see code and results, follow the link to my Github page.
